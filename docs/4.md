##  Описание прецедентов 

### 1 Регистрация пользователя

**Цель:**
Создание нового аккаунта пользователя для доступа к системе.

**Актор:**
Пользователь (студент).

**Основной сценарий:**

1. Пользователь открывает страницу регистрации.
2. Вводит регистрационные данные (логин, пароль, адрес электронной почты).
3. Подтверждает регистрацию.
4. Система проверяет корректность введённых данных.
5. Система создаёт новый аккаунт пользователя.

**Исключительные ситуации:**

* введены некорректные данные — система выводит сообщение об ошибке;
* пользователь с указанным логином уже существует — регистрация отклоняется.

---

### 2 Авторизация пользователя

**Цель:**
Предоставление пользователю доступа к функциям системы.

**Актор:**
Пользователь.

**Основной сценарий:**

1. Пользователь открывает страницу авторизации.
2. Вводит логин и пароль.
3. Система проверяет корректность введённых данных.
4. Пользователь получает доступ к системе.

**Исключительные ситуации:**

* неверный логин или пароль — система выводит сообщение об ошибке;
* пользователь заблокирован — система отказывает в доступе.

---

### 3 Просмотр учебных материалов

**Цель:**
Ознакомление пользователя с доступными учебными материалами.

**Актор:**
Пользователь.

**Основной сценарий:**

1. Пользователь выбирает учебную группу.
2. Система отображает список доступных материалов.
3. Пользователь применяет поиск или фильтры.
4. Пользователь открывает выбранный материал для просмотра.

**Исключительные ситуации:**

* материалы отсутствуют — система выводит соответствующее уведомление.

---

### 4 Загрузка учебного материала

**Цель:**
Добавление нового учебного материала в систему.

**Актор:**
Пользователь.

**Основной сценарий:**

1. Пользователь выбирает функцию загрузки материала.
2. Указывает название, описание, дисциплину и тип материала.
3. Загружает файл или добавляет ссылку.
4. Система проверяет параметры загружаемого материала.
5. Система сохраняет материал.
6. Материал становится доступным участникам учебной группы (сразу либо после модерации).

**Исключительные ситуации:**

* недопустимый формат файла — загрузка отклоняется;
* превышен допустимый размер файла — система выводит сообщение об ошибке.

---

### 5 Оценка учебного материала

**Цель:**
Формирование рейтинга учебных материалов.

**Актор:**
Пользователь.

**Основной сценарий:**

1. Пользователь открывает учебный материал.
2. Выбирает оценку.
3. Система сохраняет оценку пользователя.
4. Система пересчитывает рейтинг материала.

**Исключительные ситуации:**

* пользователь уже оценивал данный материал — повторная оценка запрещается.

---

### 6 Комментирование учебного материала

**Цель:**
Обеспечение обратной связи и обсуждения учебных материалов.

**Актор:**
Пользователь.

**Основной сценарий:**

1. Пользователь открывает страницу материала.
2. Вводит текст комментария.
3. Подтверждает отправку.
4. Система сохраняет комментарий и отображает его в списке комментариев.

**Исключительные ситуации:**

* комментарий пустой — система отклоняет отправку.

---

### 7 Отправка жалобы на материал

**Цель:**
Сообщение о некорректном или запрещённом контенте.

**Актор:**
Пользователь.

**Основной сценарий:**

1. Пользователь выбирает функцию «Пожаловаться».
2. Указывает причину жалобы.
3. Система регистрирует жалобу.
4. Жалоба передаётся модератору для рассмотрения.

**Исключительные ситуации:**

* жалоба на данный материал уже была отправлена пользователем ранее — повторная отправка запрещается.

---

### 8 Модерация учебного материала

**Цель:**
Контроль качества и соответствия учебных материалов установленным правилам.

**Актор:**
Модератор.

**Основной сценарий:**

1. Модератор открывает список жалоб или материалов на проверку.
2. Просматривает соответствующий материал.
3. Принимает решение (оставить без изменений, скрыть или удалить материал).
4. Система применяет выбранное действие и фиксирует результат.

**Исключительные ситуации:**

* у пользователя отсутствуют права модератора — система отказывает в доступе к функциям модерации.

---
